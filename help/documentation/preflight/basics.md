# Preflight basics

![Preflight](./assets/overview/hero.png){align="center"}

Preflight helps you identify opportunities to enhance your web pages before they are published. The Preflight Extension identifies opportunities by running audits on your content and displays the results in a panel so you can address them before publishing.

## Where Preflight appears

Preflight is available in different authoring environments:

* **Universal Editor** – The Preflight Extension appears in the **side rail**. Select it to start an audit of the current page.
* **Document-based authoring** - Via the Sidekick, or bookmarklet, run the preflight tooling on your previewed page content to see the list of opportunities. 
* **AEM Sites Page Editor** – Use the Preflight bookmarklet in your browser to start an audit.

For setup instructions, see [Preflight setup](./setup.md).

## Starting an audit

To run Preflight:

1. Open the page that you want to audit in your authoring environment (Universal Editor, Document-based preview, or AEM Sites Page Editor).
2. Open the Preflight panel — select the Preflight Extension from the side rail, or click the Preflight button in Sidekick.
3. Preflight analyzes the page and displays the opportunities found to enhance the page.

## Audit results 

When the audit completes, Preflight displays the opportunities it found. Each opportunity is organized by type and includes details on how to resolve the problem.

## About Preflight opportunities

Preflight evaluates multiple aspects of your content, including accessibility, metadata, links, and readability. See [Preflight opportunities](./overview.md) for a full list of available opportunity types and how to address them.
